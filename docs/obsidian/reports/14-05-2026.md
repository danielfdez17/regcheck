# Daily Report - 14-05-2026

## Features
- Added modular frontend internationalization (i18n) using `i18next` and `react-i18next`.
- Introduced namespace-based English locale files under `frontend/src/i18n/locales/en/`.
- Migrated all user-facing frontend UI strings to translation keys across auth, home, shell, theme, and playground surfaces.

## Improvements
- Added `useAppTranslation` hook for typed namespace access.
- Added `formatDateTime` and `formatNumber` helpers with locale-aware `Intl` formatting (`en` maps to `en-GB`).
- Added `LocaleSync` component to keep `document.documentElement.lang` in sync with the active locale.
- Configured `i18next-browser-languagedetector` with `regcheck-locale` localStorage key for future language switching.

## Module layout


## Namespace map

| Namespace | Contents |
|-----------|----------|
| `common` | Brand, navigation, footer, theme toggle, shared actions (edit, delete, close) |
| `auth` | Login/signup form labels, headings, submit states |
| `home` | Hero, metrics, loading states, selector unavailable |
| `playground` | Sidebar form, assessment output, checklist UI, profile note prefixes |
| `errors` | Bootstrap errors, assessment toasts, export confirmation |

## Usage

```tsx
import { useAppTranslation } from "./i18n/hooks/use-app-translation";

function MyComponent() {
  const { t } = useAppTranslation("home");
  return <h1>{t("hero.title")}</h1>;
}
```

For non-React code (async loaders, pure functions), use the i18n instance directly:

```ts
import i18n from "./i18n";
i18n.t("backend.noRules", { ns: "errors" });
```

For dates:

```ts
import { formatDateTime } from "./i18n/format";
formatDateTime(isoTimestamp);
```

## Adding a new language

1. Copy `frontend/src/i18n/locales/en/` to `locales/<lang>/` (e.g. `es`).
2. Translate JSON **values** only; keep keys unchanged.
3. Add the locale to `SUPPORTED_LOCALES` in `config.ts` and register resources in `index.ts`.
4. Add `toIntlLocale` mapping in `config.ts` if needed.
5. Enable a `LocaleSwitcher` in the navbar when more than one locale is supported.

## Notes
- Backend/API content (GDPR rules, checklist items from the API) remains in the language returned by the server.
- Only English locale files ship in this iteration; infrastructure is ready for additional languages.
