# Daily Report - 13-05-2026

## Features
- Added multi-tenant authentication with signup, login, logout, and session restore via JWT Bearer tokens.
- Signup collects first name, last name, enterprise, email, and password; each signup creates a new tenant organization and first user.
- Login accepts email and password only.
- Passwords are hashed with bcrypt before persistence; credentials travel in JSON POST bodies over HTTPS in production.
- GDPR assessments are now scoped by `tenant_id`; all GDPR API routes require authentication.

## Improvements
- Added frontend auth gate so the application is inaccessible until the user signs in.
- Added navbar user context (name, enterprise) and logout action.
- Extended `.env.example` with `REGCHECK_JWT_SECRET_KEY`.

## Notes
- Existing SQLite databases without the new `tenants`, `users`, and `compliance_assessments.tenant_id` schema must be reset before running the authenticated stack.
